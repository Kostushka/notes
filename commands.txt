1. Команда, чтобы узнать зарезервированный номер порта: 

   grep telnet /etc/services
   grep http /etc/services
   
2. Команда ifconfig - для конфигурирования сетевого интерфейса
   -a - опция означает "все интерфейсы"
   
   /usr/etc/ifconfig -a

3. Команда netstat - информация об интерфейсах системы
   -i - флаг выдает информацию об интерфейсах
   -n - флаг выдает IP адрес вместо имен хостов

   Почему MTU для интерфейса loopback, показанного в разделе "Команда netstat", установлен в 1536?
   sun % netstat -in Name Mtu Net/Dest Address Ipkts Ierrs Opkts Oerrs Collis Queue le0 1500 140.252.13.32 140.252.13.33 67719 0 92133 0 1 0 sl0 552 140.252.1.183 140.252.1.29 48035 0 54963 0 0 0 lo0 1536 127.0.0.1 127.0.0.1 15548 0 15548 0 0 0

4. ftp bsdi

5. Команда arp - посмотреть содержимое ARP кэша 
   -a - опция показывает все записи в кэше

   arp -a 

6. Команда tcpdump
   -e - посмотреть аппаратные адреса (48-битные адреса Ethernet)

7. Команда telnet - подсоединиться к серверу

8. Команда date

9. Команда connect - сохраняет имя хоста, с которым необходимо установить соединение, и номер порта на этом хосте

   connect svr4 8888

10. Команда get - UDP датаграмма посылается на порт 8888 хоста svr4

    get temp.foo - пробуем получить файл

11. Команда ping

12. Программа traceroute - позволяет нам посмотреть маршрут, по которому двигаются IP датаграммы от одного хоста к другому

    traceroute svr4
